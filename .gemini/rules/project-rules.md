# 高中物理受力分析项目规则

## 项目目标

本项目是一个面向高中生的物理力学学习网站，用交互网页动画帮助学生学习受力分析、牛顿运动定律、摩擦、斜面、连接体、滑轮和圆周运动。

项目必须强调：
1. 受力分析整体框架。
2. 从易到难的模型训练。
3. 可视化受力图。
4. 动态参数调节。
5. 清晰的解题步骤。
6. Win11 本地可运行。

## 技术栈

- React
- TypeScript
- Vite
- SVG for force diagrams
- Tailwind CSS
- KaTeX or MathJax for formulas
- Vitest for physics calculation tests

## 教学原则

- 不要只给答案，要展示思考过程。
- 每个模型必须包含：场景、受力图、步骤讲解、公式推导、交互参数、练习题。
- 所有高中生可能混淆的地方必须用提示说明。
- 向心力不能被写成一种独立真实力，要说明它是合力效果。
- 摩擦力方向必须通过相对运动或相对运动趋势判断。
- 斜面模型必须展示重力分解。
- 连接体模型必须区分整体法和隔离法。

## 代码规则

- 所有代码必须使用 TypeScript。
- 物理计算逻辑必须放在 src/physics/。
- UI 组件必须放在 src/components/。
- 课程页面必须放在 src/lessons/。
- 数据配置必须放在 src/data/。
- 不要在 React 组件中写复杂物理计算。
- 公式、单位、物理量命名必须清晰。
- 所有核心物理计算函数必须有单元测试。

## 交互规则

每个交互模型至少包含：
1. 参数滑块。
2. 动态受力图。
3. 坐标轴显示。
4. 公式面板。
5. 步骤讲解。
6. 错误易混点提示。

## 开发流程

开始写代码前必须：
1. 阅读现有目录结构。
2. 查找已有相似模型。
3. 输出实现计划。
4. 列出将修改或新增的文件。
5. 等用户确认后再编码。

完成后必须：
1. 说明修改了哪些文件。
2. 说明每个文件的作用。
3. 说明如何运行。
4. 说明如何验证。
5. 运行测试或说明无法运行的原因。
