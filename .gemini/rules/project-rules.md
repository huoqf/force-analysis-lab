
# 物理受力分析项目规则

## 技术栈（不可偏离）
React + TypeScript + Vite + SVG + Tailwind CSS + KaTeX + Vitest

## 目录约定
- 物理计算：src/physics/
- UI 组件：src/components/
- 课程页面：src/lessons/
- 数据配置：src/data/
- 题目数据：src/data/problems/[problem_id].json
- 严禁在 React 组件内写复杂物理计算

## 教学铁律（违反必拦截）
1. 向心力不是独立真实力，必须说明其为合力效果
2. 摩擦力方向必须先判断相对运动或相对运动趋势
3. 斜面模型必须展示重力分解，禁止直接代结论
4. 连接体模型必须明确区分整体法与隔离法
5. 不许只给答案，必须包含场景、受力图、步骤、公式、参数、易错点

## 交互模型必备 6 要素
参数滑块、动态受力图、坐标轴、公式面板、步骤讲解、易错点提示

## 编码流程（强制）
开始编码前先输出实现计划与改动文件清单，等待用户确认后再写代码。
完成后说明：改了什么、如何运行、如何验证、测试结果。

## 物理计算单元测试
src/physics/ 下每个核心函数必须有 Vitest 单元测试。
